01 — Logo System
The Aesop Mark
The primary mark is an architectural figure — an abstract form representing analysis, structure, and perspective. It is used as the primary brand icon across all materials. The text wordmark pairs it with "aesop" (Abril Fatface) and "Analytics" (DM Sans, spaced caps).
Primary Mark
Navy on paper. Standard use on all light backgrounds.
Reversed Mark
Paper on navy. Use on all dark/navy backgrounds.
Monochrome Mark
Ink black. Use for single-color print, embossing, or when color is unavailable.

Logo Usage Rules
✅
Always maintain clear space equal to the mark's width on all sides.
✅
Use the navy mark on paper (#FBFAF5), white, or light neutral backgrounds.
✅
Use the white/reversed mark on navy or any dark background.
✅
Minimum nav size: 32 × 32px. Minimum print size: 0.5 inch.
🚫
Do not recolor the mark to anything other than navy, paper-white, or ink-black.
🚫
Do not place the navy mark on dark backgrounds or the white mark on light backgrounds.
🚫
Do not stretch, skew, rotate, or add drop shadows to the mark.
🚫
Do not use the old "ae" ligature Unicode character as a logo substitute in new materials.

04 — Voice & Tone
How Aesop Speaks
Authoritative without being cold. Direct without being abrupt. The voice of a senior practitioner who's seen enough broken reporting environments to skip the pleasantries.
Core Positioning Statements
"I help growing companies modernize their reporting systems so leaders can trust the numbers, teams can align on KPIs, and decisions can move faster."
"Most reporting problems are not visualization problems. They are ownership, definition, process, trust, and governance problems."
"You get the expert, not a junior team."
Anti-Positioning — Aesop Is NOT
🚫 A dashboard factory or generic BI contractor
🚫 A "data guy for hire" or staff augmentation
🚫 A data engineering body shop
🚫 A tool reseller or platform partner
🚫 A vague AI consultant
🚫 An "AI-powered firm" — AI is internal leverage only
Writing Principles
📌
Lead with the business problem, not the technology. "Your teams are running the business from spreadsheets" before "Power BI can help."
📌
Name the real culprit. Reporting failures are governance, ownership, and definition problems. Not bad charts.
📌
Specific over vague. "Finance and Sales are using different definitions of revenue" beats "alignment issues."
📌
Fixed-fee and scoped. Always describe work as fixed-fee, scope-bounded, built to finish. Never hourly.
📌
Senior practitioner voice. Write as if Jonathan is in the room. First-person singular is appropriate on about/services pages.
📌
No fabricated data or fake social proof. All case studies anonymized. All testimonials real and attributed.
✓ On-brand language
"Book a diagnostic" (nav CTA — always)
"Reporting modernization & data trust consulting" (footer tagline — exact)
"Flagship" when referencing the Assessment
"Boutique consulting" for the firm
"Platform-agnostic" for tool approach
✗ Off-brand language
"Book a call" / "Get in touch" / "Start a conversation" (not the nav CTA)
"AI-powered" to describe the firm
"Tableau expert" or platform-specific identity
"Unlimited revisions" or scope-vague language
Generic BI phrases: "unlock insights", "data-driven journey"

06 — Agent Rules
AI Agent Quick Reference
Authoritative rules for any AI agent building or editing Aesop Analytics site files. These override general assumptions.

Identity & Positioning

  • Firm = reporting modernization + data trust specialist
  • NOT: generic BI, dashboard factory, AI consultant
  • NOT: AI-powered — all work is human-led
  • Flagship offer = Reporting Clarity Assessment
  • Nav CTA = "Book a diagnostic" (exact phrase)
  • Footer tagline = "Reporting modernization & data trust consulting."

Palette Rules

  • Primary dark = --navy #0F1F38 (not green)
  • Primary accent = --twine #C18C5D
  • Background = --paper #FBFAF5
  • NEVER use: #092509, #0D350D (old greens)
  • NEVER use any green-tinted dark values
  • Fix stray --green occurrences → --navy

Typography Rules

  • Display font: Abril Fatface — hero, h2, pull quotes, stats
  • Body/UI font: DM Sans — everything else
  • Abril NEVER in: nav, buttons, labels, captions, body
  • Body font-size: 18px, line-height 1.65
  • Eyebrow: DM Sans 700, uppercase, ls .28em, twine
  • Never use Lato, Helvetica, or Georgia as body font

Layout & Structure

  • Site max-width: 1080px (.wrap class)
  • Article content max-width: 740px
  • Nav order: Services · Assessment · Work · Insights · Resources · About · Contact
  • Every article links back to insights.html in footer
  • CSS load order: shared.css → core.css or article.css
  • work.html: always use Python splice scripts (BOM issues)

Logo & Mark

  • Primary mark: aesop_mark_navy_twine.svg (light bg)
  • Reversed mark: aesop_mark_white_twine.svg (dark bg)
  • Favicon: favicon.svg (in WEBSITE root)
  • Nav/footer: inline SVG mark, NOT the æ Unicode character
  • Eye color always: #C18C5D (twine)
  • Do not stretch, skew, or recolor the mark

Content Rules

  • No fabricated data, stats, or testimonials
  • Case studies: all anonymized by default
  • Pricing: fixed-fee per project, never hourly
  • Even-handed on contested economic/political topics
  • Field studies: cite primary source (BEA, EPI, SSA)
  • Framework name: TRUST (Triage/Reconcile/Unify/Standardize/Transition)

13 — Newsletter Template
The Reporting Trust Letter
Table-based HTML email template. Built for deliverability with inline styles and table layouts for Outlook/Gmail compatibility. Four zones: feature article, quick insights (3-up), diagnostic spotlight, and branded footer.
📐
Max width 600px — standard email content width, renders cleanly on mobile and desktop clients.
✏️
All copy in brackets — every [field] is a clear placeholder. Replace headline, lead paragraphs, quick insights, and diagnostic prompt per issue.
🎨
Inline styles only — required for Outlook. Never move styles to a <head> stylesheet when preparing for send.
📧
Send via Mailchimp, ConvertKit, or Substack — paste HTML into your platform's custom HTML block. Test in Litmus or Email on Acid before first issue.
🗓️
Recommended cadence: bi-weekly or monthly. One strong insight beats a thin weekly digest.
📁
File: BRANDING/06 templates/newsletter_template.html — open in browser for full preview before sending.
Content structure per issue: Header: Issue number + date — update before each send
Feature: One article (400–700 words) — link to site article or self-contained
Quick Insights: 3 short observations (2-3 sentences each)
Spotlight: One diagnostic question with brief framing — actionable for any reporting leader
Footer: Static — update unsubscribe link per platform
